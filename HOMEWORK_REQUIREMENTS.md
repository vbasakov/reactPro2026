## 1. Общие правила

### Репозиторий и ветки

- Один репозиторий на весь курс (GitVerse/GitHub).
- Ветка строго `lesson-N`, где N — номер урока (1–9).
- Итоговый проект — ветка `final` (допустимо `itog`).
- Один PR = один урок. База PR — `main`.
- В ветке не смешивать работу разных уроков.

### Ссылка на репозиторий (на платформе курса)

Указывайте **корень репозитория** — одну ссылку на весь курс. Не ветку, не папку, не PR.

| Верно | Неверно |
|-------|---------|
| `https://github.com/user/react-pro` | `.../tree/main` |
| `https://gitverse.ru/user/react-pro` | `.../tree/lesson-3` |
| | `.../pull/5` |
| | `.../blob/main/src/App.tsx` |

Перед отправкой откройте ссылку в режиме инкогнито (или попросите одногруппника) и убедитесь, что репозиторий **открывается**: виден код, ветки, список PR. Если 404 или «нет доступа» — ревьюер тоже не увидит работу.

**Приватный репозиторий** — допустим, но тогда обязательно:

1. На платформе отметьте, что репо **приватный**.
2. Добавьте проверяющего **collaborator** (достаточно read) или назначайте **reviewer** на каждый PR.
3. В комментарии к ссылке напишите, доступ уже выдан или нет.

Без доступа репозиторий для проверки считается не сданным, даже если PR открыт.

### Что должно быть в каждой ветке


| Файл / настройка                                 | Зачем                                              |
| ------------------------------------------------ | -------------------------------------------------- |
| `LESSON-N.md` в корне                            | чеклист, команды запуска, ссылки на ключевые файлы |
| `package-lock.json`                              | фиксированные зависимости                          |
| `"engines": { "node": ">=20" }` в `package.json` | единое окружение                                   |
| Скрипты `dev`, `build`, `lint`                   | стандартная проверка                               |


### Что нельзя коммитить

`node_modules/`, `dist/`, `.env`, артефакты сборки, незавершённый merge/rebase.

### Описание PR

В описании каждого PR укажите:

```
Студент: Иван Иванов, @telegram_nick
Запуск: npm ci && npm run dev
Сделано: ...
Не успел / вопросы: ...
```

Имя — как в зачётке / на платформе. Ник в Telegram — с `@`, чтобы можно было написать в лс по результатам ревью.

### Минимальная проверка перед PR

```bash
npm ci
npm run build
npm run lint   # если настроен
```

Для урока 7 дополнительно — скрипты сборки esbuild (см. раздел 10).

## 2. Формат `LESSON-N.md`

Одна страница на урок:

```markdown
# LESSON-N — <тема>
Ветка: lesson-N

## Запуск
npm ci && npm run dev
npm run build
## Чеклист
- [ ] критерий — N балл

тут лучше указывать, что сделано, т.к. ревьюер может что-то пропустить, либо студент положит файлы далеко.


## Не сделано / вопросы
- ...
```

Студент отмечает `[x]` только то, что реально сделано. Ревьюер и автоматика сверяют чеклист с кодом.

## 13. Итоговый проект - ветка `final`



### Обязательные условия (без любого - может быть незачет)

- `npm run build` / `npm run dev` проходит
- Модалка через `createPortal` + `#modal-root`, ESC и overlay
- `useActionState` или `useOptimistic` на реальном UX-кейсе
- README: что сделано, структура, сравнение сборок (цифры), скрины/гиф

### Что будем проверять

- `shared/ui` — только презентационные компоненты, без стора и API
- фичи изолированы в `features/`, слои `shared/`, `pages/` соблюдены
- оптимизация рендеров: Profiler, `React.memo`, `useMemo`/`useCallback` в списках
- портальная модалка: ESC, overlay, фокус при открытии и возврат на триггер
- `useRef` — состояние без лишних ререндеров, работа с DOM
- альтернативная сборка (esbuild/SWC/Vite) и сравнение с Webpack в README
- React 19: реальный UX-кейс с reset или оптимистичным UI

Файл сдачи: `FINAL.md` по аналогии с `LESSON-N.md`.

## 15. Типичные ошибки, которые лучше предусмотреть, т.к. вижу их часто

- В описании PR нет имени и ника в Telegram
- Ссылка на ветку, файл или PR вместо корня репозитория
- Приватный репо без доступа проверяющему
- Ветка не `lesson-N` / в PR код двух уроков
- Нет `LESSON-N.md` или чеклист пустой
- `npm ci && npm run build` падает
- Нет `package-lock.json` или `engines.node`
- В коммите `node_modules` / `dist`
- Урок 7: сборка не в `build-hw/`, нет скриптов в `package.json`
- Урок 6: токен читается напрямую из localStorage в `ProtectedRoute`, минуя контекст
